# sendCMD 導入手順書 (生徒用PC・サービス側)

このドキュメントは、管理対象となる生徒用PCに `sendCMD` サービスを導入し、常駐させるための手順書です。

---

## 1. 動作環境と事前準備
*   **対象OS:** Windows 10 / Windows 11 (64bit)
*   **実行権限:** Windows サービスとして登録するため、**管理者権限**が必要です。
*   **通信要件:** 教員用PCからアクセス可能なローカルネットワーク環境。生徒PC側の **TCP 5000番ポート** を開放する必要があります。
*   **時刻同期要件:** セキュリティ（HMAC-SHA256署名認証）の仕様上、教員PCと生徒PCの間で**システム時刻のずれが5分（300秒）以内**である必要があります。時刻が大きくずれていると通信が自動的に拒否されるため、必要に応じてNTPサーバー等による自動時刻同期を設定してください。

---

## 2. 導入手順

### ステップ 1: 設定の編集 (appsettings.json)
配布された `server` フォルダ内の `appsettings.json` をテキストエディタで開き、環境に合わせて設定値を変更します。
*   **`ApiKey`**: 教員PCと通信する際の共有秘密鍵（認証暗号キー）です。送信時はこのキーを用いてリクエストごとに HMAC-SHA256 署名が自動生成され、ネットワーク上にはキー自体は平文で流れません。任意の安全な文字列に変更してください（教員PC側と一致させる必要があります）。
*   **`UploadDirectory`**: 教員PCから送られてきたインストーラーを一時的に保存するフォルダです。**デフォルトは `C:\Users\Public\sendCMD_uploads` です。**
    *(※ SYSTEM権限と一般の対話型ユーザーセッション（生徒ユーザー）の双方がアクセス・実行可能な Public パスを使用することが推奨されます)*

```json
{
  "ApiKey": "設定した独自の暗号キー（教員側と共通）",
  "UploadDirectory": "C:\\Users\\Public\\sendCMD_uploads"
}
```

### ステップ 2: install.bat の実行（自動セットアップ）
1.  配布ファイル内の **`install.bat` をダブルクリック** (または右クリックして「管理者として実行」) します。
2.  「ユーザーアカウント制御 (UAC)」のダイアログが表示されるので、「はい」をクリックして許可します。
3.  自動的に以下の処理が行われます：
    *   `C:\Program Files\sendCMD` フォルダの作成とファイルのコピー
    *   Windows ファイアウォール規則の追加 (TCP 5000番ポートの受信許可)
    *   Windows サービスへの登録と開始
4.  コマンドプロンプト画面に「セットアップが正常に完了しました！」と表示されたら、キーを入力して画面を閉じます。

これで導入は完了です。

---

### アンインストールについて
サービスを完全に削除したい場合は、同様に配布ファイル内の **`uninstall.bat` をダブルクリック** して実行してください。サービスの停止、登録削除、ファイアウォール規則の削除、ファイルの削除がすべて自動で行われます。

---

## 3. サービスの管理コマンド

サービスの停止や削除を行いたい場合は、管理者用PowerShellで以下のコマンドを使用します。

*   **サービスの停止:**
    ```powershell
    sc.exe stop sendCMD
    ```
*   **サービスの再起動（設定ファイル反映時など）:**
    ```powershell
    sc.exe stop sendCMD
    sc.exe start sendCMD
    ```
*   **サービスのアンインストール（削除）:**
    ```powershell
    sc.exe stop sendCMD
    sc.exe delete sendCMD
    ```

---

## 4. ログファイルと動作状況の確認
サービスや常駐ヘルパープロセスの動作状況、および実行されたコマンドの監査履歴は、生徒PCの以下のログファイルに記録されます。
*   **ログファイルパス**: `C:\Users\Public\sendCMD_server_log.txt`
*   **ログの記録内容**:
    *   サービスの起動・ヘルパープロセスの常駐開始ログ
    *   外部プロセスの起動失敗などのエラーログ
    *   **実行コマンド監査ログ**: 教員側から送られてきたPowerShellコマンド実行要求 (`/api/exec`) の実行開始時刻、セッションタイプ、およびコマンド本文。
    *   **ファイルアップロードログ**: 教員側から送られてきたファイルのアップロード成功時の保存先パスとファイルサイズ。
    *(※ 5秒間隔のアクティブアプリ取得やプロセス一覧取得などの定期監視APIに関しては、ログファイルの不要な肥大化を防ぐため、正常時は記録されません)*

## 5. API認証キーの変更・更新手順

セキュリティ向上のため、あるいは定期的なメンテナンスでAPI認証キーを変更する際は、以下のいずれかの方法で生徒PC側と教員PC側の双方を更新してください。

### 💡 方法A: 教員PCからリモートで一括更新する（推奨）

生徒PCの台数が多い場合、教員アプリの「ファイル配布とインストール」機能を使用して、全PCに対してリモートから一括で設定変更・サービス再起動を行うことができます。

1.  テキストエディタを開き、以下の内容を記述した PowerShell スクリプトファイル（例: `update_key.ps1`）を作成して保存します（※ `"新しい暗号キー"` の部分を設定したいキーに書き換えてください）。
    ```powershell
    $path = "C:\Program Files\sendCMD\appsettings.json"
    if (Test-Path $path) {
        $json = Get-Content $path -Raw | ConvertFrom-Json
        $json.ApiKey = "新しい暗号キー"
        $json | ConvertTo-Json -Depth 10 | Out-File $path -Encoding utf8
        # レスポンスの切断を防ぐため、2秒遅延させてバックグラウンドでサービスを再起動します
        Start-Process powershell.exe -ArgumentList "-NoProfile -Command 'Start-Sleep -Seconds 2; Restart-Service sendCMD -Force'" -WindowStyle Hidden
    }
    ```
2.  教員アプリ（`client.exe`）を起動し、画面上部の「**API認証キー**」欄には**現在の（変更前の）キー**を入力した状態にします。
3.  「**ファイル配布とインストール**」タブを選択し、作成した `update_key.ps1` を指定します。
4.  **「ログインユーザー権限で実行する」** チェックボックスを必ず **オフ** にします（オフにすることで、管理者権限である SYSTEM 権限でスクリプトが実行されます）。
5.  「**ファイルを配布してインストール**」を実行します。
6.  実行完了後、生徒PC側のサービスが順次自動で再起動します。
7.  最後に、教員アプリ側の「**API認証キー**」欄に入力されているキーを「**新しい暗号キー**」に書き換えます。これで一括更新は完了です。

### 🛠️ 方法B: 生徒PCで直接個別に更新する

直接生徒PCを操作して変更する場合の手順です。

1.  生徒PC上のインストールフォルダ（デフォルト: `C:\Program Files\sendCMD\`）にある `appsettings.json` を管理者権限のテキストエディタで開きます。
2.  `ApiKey` の値を新しいキー文字列に変更して保存します。
3.  **サービスの再起動（必須）**: 新しいキーをメモリに読み込ませるため、管理者権限のPowerShellを開き、以下のコマンドを実行してサービスを再起動します。
    ```powershell
    sc.exe stop sendCMD
    sc.exe start sendCMD
    ```
    *(※ サービスが再起動されるまで、生徒PCは古いキーまたはエラー状態で動作します)*

### ステップ 2: 教員PC側（クライアント）のキー更新
1.  教員PCで `client.exe` を起動します。
2.  画面上部の「**API認証キー**」欄に、ステップ1で生徒PC側に設定した新しいキーを入力します。
3.  入力された新しいキーは、アプリと同じフォルダにある `config.json` ファイルに自動的に保存（永続化）されます。次回以降の起動時は自動的にこのキーがロードされるため、再入力の必要はありません。
