# sendCMD 導入手順書 (生徒用PC・サービス側)

このドキュメントは、管理対象となる生徒用PCに `sendCMD` サービスを導入し、常駐させるための手順書です。

---

## 1. 動作環境と事前準備
*   **対象OS:** Windows 10 / Windows 11 (64bit)
*   **実行権限:** Windows サービスとして登録するため、**管理者権限**が必要です。
*   **通信要件:** 教員用PCからアクセス可能なローカルネットワーク環境。生徒PC側の **TCP 5000番ポート** を開放する必要があります。

---

## 2. 導入手順

### ステップ 1: 設定の編集 (appsettings.json)
配布された `server` フォルダ内の `appsettings.json` をテキストエディタで開き、環境に合わせて設定値を変更します。
*   **`ApiKey`:** 教員PCと通信する際の認証暗号キーです。任意の安全な文字列に変更してください（教員PC側と一致させる必要があります）。
*   **`UploadDirectory`:** 教員PCから送られてきたインストーラーを一時的に保存するフォルダです。**デフォルトは `C:\Users\Public\sendCMD_uploads` です。**
    *(※ SYSTEM権限と一般の対話型ユーザーセッション（生徒ユーザー）の双方がアクセス・実行可能な Public パスを使用することが推奨されます)*

```json
{
  "ApiKey": "設定した独自の認証キー",
  "UploadDirectory": "C:\\Users\\Public\\sendCMD_uploads"
}
```

### ステップ 2: install.bat の実行（自動セットアップ）
1.  配布ファイル内の **`install.bat` をダブルクリック** (または右クリックして「管理者として実行」) します。
2.  「ユーザーアカウント制御 (UAC)」のダイアログが表示されるので、「はい」をクリックして許可します。
3.  自動的に以下の処理が行われます：
    *   `C:\Program Files\sendCMD` フォルダの作成とファイルのコピー
    *   Windows ファイアウォール規則の追加 (TCP 5000番ポートの受信許可)
    *   Windows サービスへの登録と開始
4.  コマンドプロンプト画面に「セットアップが正常に完了しました！」と表示されたら、キーを入力して画面を閉じます。

これで導入は完了です。

---

### アンインストールについて
サービスを完全に削除したい場合は、同様に配布ファイル内の **`uninstall.bat` をダブルクリック** して実行してください。サービスの停止、登録削除、ファイアウォール規則の削除、ファイルの削除がすべて自動で行われます。

---

## 3. サービスの管理コマンド

サービスの停止や削除を行いたい場合は、管理者用PowerShellで以下のコマンドを使用します。

*   **サービスの停止:**
    ```powershell
    sc.exe stop sendCMD
    ```
*   **サービスの再起動（設定ファイル反映時など）:**
    ```powershell
    sc.exe stop sendCMD
    sc.exe start sendCMD
    ```
*   **サービスのアンインストール（削除）:**
    ```powershell
    sc.exe stop sendCMD
    sc.exe delete sendCMD
    ```
