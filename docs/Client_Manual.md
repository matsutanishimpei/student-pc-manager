# sendCMD 操作手順書 (教員・クライアント側)

このドキュメントは、教員用PCから生徒用PCに対してコマンドを実行したり、ソフトウェアを配布・インストールしたりするための操作手順書です。

---

## 1. 起動方法
1.  配布ファイル内の `client.exe` をダブルクリックして起動します。
    *   このアプリは単一ファイルで動くため、インストールは不要です。

---

## 2. 画面説明と操作手順

### ① ターゲットPC（生徒PC）の管理

操作対象とする生徒PCを登録・管理します。本システムでは、IPアドレスの直接指定だけでなく、**名前解決（DNS / LLMNR）に対応しているため、「PC名 (ホスト名)」を直接登録して通信することが可能**です。これにより、DHCP環境でIPアドレスが変わっても再設定の必要がありません。

#### 個別に登録する場合:
1.  左上のテキストボックスに、生徒PCの IPアドレス または **PC名** と、ポート番号（例: `PC-ROOM1-01:5000`）を入力します。
2.  「**追加**」ボタンをクリックすると、下のリストに追加されます。

#### PCの命名規則から一括で自動生成する場合:
実習室のPC名が規則的（例: `PC-STUDENT-01` 〜 `PC-STUDENT-40`）な場合は、以下の手順で一括生成できます。
1.  「**PC名の一括生成**」パネルをクリックして展開します。
2.  **接頭辞** (例: `PC-STUDENT-`)、**開始番号** (例: `1`)、**終了番号** (例: `40`)、**桁数** (例: `2` 桁なら `01` 形式、`1` 桁なら `1` 形式)、**ポート番号** (例: `5000`) を指定します。
3.  「**一括生成**」ボタンをクリックすると、指定範囲のPC名が一斉に生成され、重複を除いてリストに追加されます。

#### 選択と削除の操作:
*   リスト内のチェックボックスをオンにすることで、操作対象を指定します（複数選択・全台選択可）。
*   「**全選択**」および「**全解除**」ボタンで、一括チェックの切り替えが可能です。
*   チェックが入っているPCをリストから削除したい場合は、「**削除**」ボタンをクリックします（確認ダイアログが表示されます）。

#### 学生名の割り当てと名簿CSV連携:
PCのIPアドレスやマシン名の代わりに、座席表や名簿の学生名を表示することができます。
*   **個別登録**: リスト内のPCを右クリックし、「**学生名を割り当て...**」を選択してダイアログに名前を入力します。解除する場合は「**割り当て解除**」を選択します。学生名が設定されると、リスト上では `学生名 (IPアドレス)` の形式で優先表示されます。
*   **名簿CSVのインポート**: 「**学生名 ▼**」ボタンから「**CSVからインポート...**」を選択します。`IPアドレス/PC名,学生名` の形式で書かれたCSVファイルを読み込むことで、一括でマッピング・上書きできます。読み込み前に既存の割り当てをクリアするかも選択できます。
*   **名簿CSVのエクスポート**: 「**学生名 ▼**」ボタンから「**CSVへエクスポート...**」を選択します。現在のPCリストと学生名の紐付け状態を、インポート互換のあるCSV形式で保存可能です（Excel等で直接開けるように `CP932 (Shift-JIS)` エンコーディングで出力されます）。

#### 操作ログの永続化保存:
*   コンソール上で行った各種操作や通信ログは、アプリを再起動しても履歴を辿れるよう、以下のローカルログファイルに自動的に日時付きで追記保存されます。
    *   `%LOCALAPPDATA%\sendCMD\client_operation_log.txt`

> [!TIP]
> **リストの自動保存:** 
> 登録・生成したPCリストは、アプリと同じフォルダにある `pcs.json` ファイルに自動的に保存されます。次回以降のアプリ起動時には自動でロードされるため、再度入力する必要はありません。

### ② 認証設定
1.  画面上部の「**API認証キー**」欄に、生徒PC側の `appsettings.json` に設定した認証キーを入力します。
    *   キーが一致しない生徒PCへの要求は、セキュリティ上自動的に拒否されます。

### ③ PowerShell コマンドの直接実行
1.  「**PowerShell 実行**」タブを選択します。
2.  テキストエリアに、生徒PCで実行したいPowerShellコマンドを入力します。
    *   **実行権限の仕様**: 
        本機能に入力されたコマンドは、デフォルトで**「現在サインインしている生徒の対話型ユーザーセッション（ログインユーザーの権限）」上で安全に実行されます。** これにより、生徒が動かしているアプリの操作や情報収集、あるいはそのユーザー環境特有のPowerShell処理（`whoami`で生徒のユーザー名を返す等）が可能です。
    *   例1 (PCのシャットダウン): `Stop-Computer -Force`
    *   例2 (特定プロセスの強制終了): `Stop-Process -Name "notepad" -Force`
    *   例3 (現在ログインしているユーザー名の確認): `whoami`
3.  「**選択されたPCで実行**」をクリックすると、選択中の全PCへ非同期（並列）で指示が送信され、実行結果が下の「**実行ログ・ステータス**」にリアルタイム表示されます。

### ④ ソフトウェア（インストーラー）の配布とインストール
生徒PCに対して、インストーラーファイルを送信し、画面に表示させずにサイレントインストールします。
1.  「**ファイル配布とインストール**」タブを選択します。
2.  「**参照...**」ボタンをクリックし、配布したいインストーラーファイルを選択します（`.msi` , `.exe` , `.msix` , `.appx` など）。
3.  「**アップロード後にこのファイルを実行する**」チェックボックスがオンになっていることを確認します。
4.  **「ログインユーザー権限で実行する」** チェックボックス（手動切り替え）：
    *   **通常（オフ）**: `.msi` や `.exe` は PC の最高管理者権限（`SYSTEM` 権限）で裏でインストールされます。
    *   **オン**: 一般ユーザー（生徒）のデスクトップセッションでインストーラーを起動します（生徒の `AppData` 配下にのみインストールする Discord や VS Code User インストーラーなどの配布に有効です）。
    *   *(※ `.msix` や `.appx` ファイルの場合は、チェックに関わらず自動的にログインユーザーのプロファイル領域へインストールされます)*
5.  「**実行時引数**」欄に、サイレントインストール用のパラメータを入力します。
    *   MSIファイル（`.msi`）の場合: デフォルトで入力されている `/qn`（画面を表示せずサイレント実行する標準引数）で動作します。
    *   EXEファイル（`.exe`）の場合: ソフトウェアメーカーが指定するサイレント引数（例: `/S` や `/silent` や `/verysilent` など）に変更します。
6.  「**ファイルを配布してインストール**」をクリックします。
    *   最大500MBまでのファイルに対応しており、バックグラウンドでアップロード処理と実行処理が順番に行われ、処理結果がログに出力されます。詳細は [file_distribution_spec.md](file:///d:/dev/sendCMD/docs/file_distribution_spec.md) を参照してください。

### ⑤ 稼働監視 (リアルタイム) とプロセス強制終了
生徒PCで何が動いているかを一括監視し、必要に応じて特定のタスクを強制終了します。
1.  「**稼働監視 (リアルタイム)**」タブを選択します。
2.  「**一斉取得**」ボタンをクリックします。
    *   選択された全PCに対して非同期で問い合わせを行い、各PCで生徒が現在使用している「アクティブアプリ（ウィンドウ）」の一覧を表示します。
3.  **特定のPCのプロセスを詳細に調査・終了する場合:**
    *   監視リスト内の該当するPCの行を**ダブルクリック**します。
    *   専用の「プロセス詳細監視」ウィンドウが開き、そのPCで実行されているすべてのプロセス名、PID、ウィンドウタイトルが一覧表示されます。
    *   終了させたい迷惑なプロセス（ゲームなど）を見つけたら、その行の「**強制終了**」ボタンをクリックします。
    *   確認メッセージで「はい」を選択すると、そのPCのプロセスが子プロセスもろとも強制終了（`taskkill /f /t`）され、一覧が自動更新されます。これにより、ブラウザのタブプロセスなどがゾンビ化して残るのを確実に防止します。
    *   **プロセスの除外リストへの登録**:
        *   監視リスト内で「常に非表示にしたいプロセス名」（常駐システムプロセスなど）を見つけた場合、右クリックして「**このプロセス名を除外リストに追加**」を選択することで、そのプロセスを非表示にすることができます。
        *   「**除外設定...**」ボタンをクリックすると、ローカルの除外リストファイル（`client_exclude_processes.txt`）がテキストエディタで直接開かれ、手動で除外パターンを編集・追加できます。

### ⑥ IPスキャン (生徒PCの自動検出)
ローカルネットワーク上のアクティブなPCを一斉にスキャンし、ターゲットPCリストにまとめて追加することができます。
1.  左側のサイドバーで「**IPスキャン**」タブを選択します。
2.  **開始IP** (例: `192.168.1.1`)、**終了IP** (例: `192.168.1.254`)、および待ち受け**ポート** (通常は `5000`) を入力します。
3.  「**IP範囲スキャン開始...**」ボタンをクリックします。
    *   スキャンは高速な並列処理により、通常数秒〜数十秒で完了します。
    *   生存しているPCが検出されるとリストに表示され、PC名が取得できたものは最初からチェックボックスにチェックが入ります。
4.  追加したいPCにチェックが入っていることを確認し、「**選択したPCを登録**」ボタンをクリックします。これにより、メインのターゲットPCリストに一括で登録されます。

## 3. 推奨システム規模と対応PC数の目安 (スケーラビリティ)

本システムは、教員コンソール側および生徒用サービス側の双方で、.NET 8 の高度な非同期スレッド（Task Parallelism）およびソケット再利用（HttpClient シングルトン設計）を採用しているため、非常に軽量かつスケールしやすい設計になっています。
一般的な運用規模および限界性能の目安は以下の通りです。

### ① 推奨運用規模 (1システムあたり)
*   **推奨PC数**: **40台 〜 120台** (標準的なPC教室1〜2室分)
*   **実用最大PC数**: **500台以上** (通信スペック・UI表示上の上限はありません)

### ② 各操作におけるパフォーマンス・帯域負荷の目安
ネットワーク環境が一般的な **1Gbps 有線LAN** の場合における、PC台数ごとの推定所要時間です。

| 操作内容 | 40台 (標準) | 100台 (中規模) | 500台 (大規模) | ボトルネック要因 |
| :--- | :--- | :--- | :--- | :--- |
| **PowerShell実行**<br>(軽量コマンド) | 約 1.0 秒以下 | 約 1.5 秒以下 | 約 3.0 秒以下 | 各PCのPowerShell起動速度 |
| **稼働監視・プロセス一斉取得** | 約 0.5 秒以下 | 約 0.8 秒以下 | 約 2.5 秒以下 | 各PCの内部API応答速度 |
| **画面キャプチャ一斉取得** | 約 2.5 秒 | 約 4.5 秒 | 約 20.0 秒 | ネットワーク帯域 (1枚約100KB) |
| **ファイル配布 (10MBの配布)** | 約 2.0 秒 | 約 4.0 秒 | 約 20.0 秒 | 教員PCのアップリンク送信帯域 |
| **ファイル配布 (100MB of setup)** | 約 15.0 秒 | 約 40.0 秒 | 約 3.5 分 | 教員PCのアップリンク送信帯域 |
| **ファイル配布 (500MB of setup)** | 約 1.5 分 | 約 3.5 分 | 約 18.0 分 | 教員PCのアップリンク送信帯域 |

### ③ 大規模運用時 (100台以上) のTips・注意点
*   **ファイル配布時のネットワーク帯域**:
    教員用アプリは、複数PCに対して完全に並行（パラレル）でファイルをアップロードします。100MB以上の巨大なファイルを100台以上に同時に配布する場合、教員PCのネットワークカード (1Gbpsなど) の帯域上限までデータを送信するため、一時的に教員PCの通信負荷が最大になります。数GB規模のソフトを一斉に配る際は、グループ機能等を用いて「20台ずつに分けて実行する」などの運用が推奨されます。
*   **CPU・メモリ使用量**:
    教員用アプリのメモリ使用量は、500台のPCを登録した状態でも **50MB 〜 80MB** 程度と非常にコンパクトです。CPU使用率もコマンド送信時以外はほぼ 0% で推移します。

## 4. トラブルシューティング

*   **ログに「エラー: 接続を拒否されました」または「タイムアウト」と表示される場合:**
    *   生徒PCのIPアドレスおよびポートが正しいか確認してください。
    *   生徒PC側の `sendCMD` サービスが起動しているか確認してください（`sc.exe query sendCMD`）。
    *   生徒PC側のファイアウォールで 5000 番ポートが開放されているか確認してください。
*   **ログに「Unauthorized: Invalid or missing API Key」と表示される場合:**
    *   教員PCアプリ上の「API認証キー」欄に入力した値と、生徒PC側の `appsettings.json` のキーの値が一致しているか確認してください。
*   **インストーラーを実行したが、実際にはインストールされていない場合:**
    *   インストーラーの「実行時引数」がサイレントインストール用として正しいか、確認してください。
    *   **MSIX/APPX の場合**: すでに同じバージョンのパッケージがインストールされていると、Windowsの仕様により**一切エラーを出さずに一瞬で処理をスキップ終了（ExitCode: 0）します。** 再インストールを検証する場合は、教員コンソールのPowerShell実行タブから `Remove-AppxPackage -Package <パッケージ名>` を流して一度削除してください。
    *   生徒PC側のサービス権限（SYSTEM）では、インストーラーによってネットワークドライブ上のリソースにアクセスできない場合があります。インストーラー自体はローカルフォルダ（`C:\Users\Public\sendCMD_uploads`）にコピーされるため、インストーラー内部でネットワークリソースを参照するロジックがないか確認してください。
