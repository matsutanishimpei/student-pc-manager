# sendCMD 操作手順書（教員PC）

この文書は、教員用アプリ`client.exe`の導入と日常操作を説明します。管理対象は、同じ教室内ネットワークにある生徒PCです。

## 1. 利用前の準備

- 生徒PCにsendCMDサービスが導入・起動済みであること
- 教員PCから生徒PCのTCP 5000番ポートへ接続できること
- 教員PCと生徒PCに同じAPIキーが設定されていること

本システムは意図的にHTTPSを使用せず、全生徒PCで同じAPIキーを使用します。信頼済みの閉じた教室LANだけで利用してください。APIキー漏えいまたはプロビジョニングUSB紛失時は、教員クライアントと全生徒PCのキー交換が必要です。詳細は[セキュリティ設計判断](Security_Design_Decisions.md)を参照してください。
- Windowsの時刻同期が有効で、PC間の時刻差が5分以内であること

`client.exe`を起動し、画面上部の「API認証キー」へ生徒PCと同じ16文字以上のキーを入力します。キーはリクエストごとの署名に使われ、ネットワークへ平文では送信されません。

## 2. PCの登録

### 個別登録

PC名またはIPアドレスを入力し、追加ボタンで登録します。PC名を使う場合は、教員PCから名前解決できる必要があります。

### 一括登録

PC名の接頭辞、開始番号、終了番号、桁数を指定して連番を作成できます。登録後は意図した範囲か確認してください。

### IPスキャン

IPスキャン画面で教室内の範囲を指定すると、応答するPCを検索して追加できます。広すぎる範囲は検索時間とネットワーク負荷が増えるため、教室のサブネット内に限定してください。

### 自動保存

PC一覧は`client.exe`と同じフォルダの`pcs.json`へ、設定は`config.json`へ自動保存されます。APIキーはWindowsユーザー単位で暗号化され、設定と操作ログは現在の教員ユーザー、Administrators、Local Systemだけがアクセスできます。旧版の平文設定は初回読込時に自動移行されます。

## 3. 学生名とCSV

「学生名」メニューからCSVを読み込み、PCと学生名をまとめて対応付けできます。先頭2列は次の形式です。

```csv
PC-STUDENT-01,山田 太郎
PC-STUDENT-02,"鈴木, 花子"
```

- UTF-8 BOM付き、UTF-16 LE/BE、UTF-8、CP932を自動判定します。
- カンマを含む値、二重引用符、二重引用符のエスケープに対応します。
- 出力CSVはWindows版Excelで直接開きやすいCP932です。
- 読み込み後に日本語名と割り当て件数を確認してください。

## 4. PowerShell実行

1. 対象PCを選択します。
2. 「PowerShell実行」タブへ、実行したいPowerShellの内容だけを入力します。
3. 実行ボタンを押し、PCごとの結果をログで確認します。

`powershell.exe -Command`のような起動部分は入力不要です。通常はSYSTEM権限で実行され、入力できるコマンドは最大32,768文字です。

```powershell
Get-Process | Where-Object { $_.MainWindowTitle } |
    Select-Object ProcessName, Id, MainWindowTitle
```

強い権限で実行されるため、削除、再起動、設定変更を伴うコマンドは、代表PC1台で確認してから一括実行してください。

## 5. ファイル配布とインストール

1. 対象PCを選択します。
2. 「ファイル配布とインストール」タブでファイルを選びます。
3. 必要なサイレント実行引数を指定します。
4. 実行する権限を確認します。
5. 配布を開始し、各PCの結果を確認します。

初期アップロード上限は500 MiBです。超えるファイルを扱う場合は、生徒PC側の`MaxUploadBytes`を変更し、空き容量と所要時間も事前に確認してください。

- MSI: 通常はSYSTEM権限で実行し、初期引数は`/qn`です。
- MSIX/APPX: ログイン中のユーザーで`Add-AppxPackage`を実行します。
- EXE: 通常はSYSTEM権限です。製品ごとの無人実行引数が必要です。
- 「ログインユーザー権限で実行する」: ユーザー単位のEXEやスクリプトで使用します。

詳細な分岐と引数例は[ファイル配布仕様](file_distribution_spec.md)を参照してください。

## 6. 稼働監視とプロセス操作

「稼働監視」タブでは、オンライン状態と利用中アプリを定期取得します。PC行をダブルクリックすると、画面キャプチャとプロセス一覧を確認できます。

プロセス終了は未保存データを失う可能性があります。PCとPIDを確認してから実行してください。監視対象外にしたい常駐プロセスは、右クリックから除外一覧へ追加できます。

## 7. Wake on LAN

MACアドレスを取得・保存済みのPCには、Wake on LANを送信できます。生徒PCのBIOS/UEFI、ネットワークアダプター、スイッチ側でWake on LANが有効である必要があります。

## 8. ログと保存場所

- 操作ログ: `%LOCALAPPDATA%\sendCMD\client_operation_log.txt`
- PC一覧: `client.exe`と同じフォルダの`pcs.json`
- 設定: `client.exe`と同じフォルダの`config.json`

操作ログはBOMなしUTF-8で追記されます。問い合わせ時は、発生時刻、対象PC、画面の結果と合わせて確認してください。APIキーはログへ記録されません。

## 9. トラブルシューティング

### 接続できない

- 生徒PCの電源と`sendCMD`サービスを確認します。
- PC名が教員PCから名前解決できるか確認します。
- TCP 5000番のファイアウォール規則を確認します。
- IPアドレスが変わっている場合は再スキャンします。

### 認証エラーになる

- 教員PCと生徒PCのAPIキーが完全に一致しているか確認します。
- 生徒PCの`ApiKey`が空でないことを確認します。空の場合、APIはすべて拒否されます。
- 両PCの日時、タイムゾーン、Windows Timeサービスを確認します。時刻差が5分を超えると拒否されます。
- キー変更後は生徒PCのサービスを再起動します。

### CSVが文字化けする

- 対応形式はUTF-8、UTF-16、CP932です。
- Excelへ渡す場合は、アプリから出力したCP932のCSVを使います。
- それ以外の文字コードは、UTF-8へ変換してから読み込みます。

### 配布に失敗する

- ファイルが500 MiB以下か、設定した上限以下か確認します。
- 生徒PCの`UploadDirectory`に十分な空き容量があるか確認します。
- EXEのサイレント引数が製品に合っているか確認します。
- ユーザー単位のアプリは「ログインユーザー権限で実行する」を使用します。
- 生徒PCのサーバーログを確認します。

## 10. 運用開始前チェック

1. 代表PC1台で、接続、軽いコマンド、監視、配布を確認します。
2. 日本語、カンマ、引用符を含むCSVを入出力します。
3. アプリ再起動後にPC一覧と設定が復元されることを確認します。
4. 実運用台数で一括操作し、応答時間とネットワーク負荷を確認します。
5. 停止中のPCを混ぜても、他PCの処理が続くことを確認します。
6. 実際に配る最大サイズに近いファイルで配布時間を測ります。
7. 教員側と生徒側のログを回収できることを確認します。
